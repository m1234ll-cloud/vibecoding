    - Context Menu > 터미널에서 열기

```powershell
 > git clone https://github.com/m1234ll-cloud/vibecoding.git
```

- 이전 작업 폴더 내 전체 폴더/파일을 복사하는 깃허브 리포지토리 폴더로 이동

##### 4.4 GitHub 푸시(업로드)

#### 5. 추가 내용

##### 5.1. AI에게 프로젝트 먼저 분석시키기

- 구현된 프로젝트를 분석
    - 개선점, 문제점, 오류(예외 포함) AI에 의해서 발견
    - 새로운 프로젝트를 구현 지시

##### 5.2. Plan Mode

- 바로 코딩하지 않고 먼저 계획을 세운 뒤 진행하는 방식
    - 프롬프트 채팅 창에서 + 클릭 후 목표 선택
    - 목표에 해당하는 프롬프트를 작성

##### 5.3. 작업 단위 쪼깨기

- 실수

```markdown
쇼핑몰 만들어줘
```

- 실제 예

```markdown
React로 쇼핑몰 프론트엔드를 만들거야.

- 1단계
 회원가입 화면 작업
- 2단계
 로그인 화면 작업
- 3단계
 상품 목록 화면
...

단계별로 진행해줘
```

##### 5.4. 컨텍스트 관리

- 컨텍스트가 계속 증가하면 AI도 이전 내용을 지워버림

```markdownd
PRD.md를 다시 읽어줘
소스코드 부분을 다시 읽어서 분석해줘
```

##### 5.5 하네스 엔지니어링

-  AI가 만든 프로그램이 제대로 동작하는지 AI가 자동으로 확인해주는 검증 방법

```
AI가 코드 생성 > 자동 실행 > 자동 테스트 > 오류 발견 > AI가 수정 > 다시 테스트
```

- PRD.md 작성방법

```markdown
현재 프로젝트를 테스트 할 수 있는 테스트 하네스를 진행해줘

조건
- Pytest 사용
- 주요기능 테스트
- 예외 텍스트 100회

AI가 수정할 때마다 자동으로 실행되는 테스트 환경을 만들어줘
```

- 테스트 환경 생성 언어별 테스트 환경 기술
    - Python : pytest
    - Java : JUnit
    - C# - xUnit, MTest, NUnit...
    - javascript : Vitest, Playwright
